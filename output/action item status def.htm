action item status (Project Management and Capital Budgeting)

A description of where the action item is in its lifecycle from when it is created to when it is closed. An action item's status is important to tracking, managing, and reporting on the work for projects. For example, the status of action items can be used to automatically mark a work package as completed, so having up-to-date action status is critical to tracking a project's progress.

Following are the action item statuses typically used by Project Management:

  • Created
  • Requested
  • Approved
  • Rejected
  • Scheduled
  • Cancelled
  • In Progress
  • In Progress - On Hold
  • Stopped
  • Completed
  • Completed - Verifies
  • Closed
